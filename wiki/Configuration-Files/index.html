Configuration

When using the Pipeline as a service, especially when integrating it in a custom setup, you probably want to configure one or two things. Two files are intended to be edited by you for this purpose. Where on the file system these files are located depends on the distribution.

config-logback.xml

With this file you can control the logging: to which files log messages are written, log rolling, the level of details that are logged, which log messages show up in the user interface, etc.
